## 2. Динамическое ценообразование промо-размещений

Цена промо-размещения вакансии рассчитывается автоматически.

### 2.1 Формула

```
ЦЕНА = БАЗОВАЯ × ГРЕЙД_МНОЖ × ДЕФИЦИТ_МНОЖ × ПОЛНОТА_СКИДКА
```

### 2.2 Компоненты

**Базовая цена:** 1 500 руб.

**Грейд-множитель** (чем выше грейд → дороже, т.к. ценнее для работодателя):

| Грейд | Множитель |
|-------|-----------|
| Intern | 0.5 |
| Junior | 0.7 |
| Middle | 1.0 |
| Senior | 1.5 |
| Lead | 2.0 |
| Principal / Staff | 2.5 |

**Дефицит-множитель** (соотношение вакансий к кандидатам в нише):

```
ratio = вакансий_в_нише / кандидатов_в_нише

ratio < 0.5  → 0.8  (избыток вакансий мал, кандидатов много)
ratio 0.5–1  → 1.0  (баланс)
ratio 1–2    → 1.3  (умеренный дефицит кандидатов)
ratio 2–3    → 1.6  (сильный дефицит)
ratio > 3    → 2.0  (острый дефицит)
```

"Ниша" определяется как: `positionCategory + positionSubcategory + grade`
Данные берутся из текущей базы вакансий и кандидатов.

**Полнота вакансии** (скидка за качество данных — мотивирует заполнять подробно):

| Условие | Скидка |
|---------|--------|
| Указана зарплатная вилка (salaryFrom + salaryTo) | -15% |
| Указан стек ≥ 3 инструментов (minRequirements) | -10% |
| Указан формат работы + город | -5% |
| **Максимальная скидка** | **-30%** |

### 2.3 Примеры расчёта

**Middle Frontend, Москва, зарплата указана, стек подробный:**
```
1500 × 1.0 (Middle) × 1.3 (дефицит) × 0.75 (скидка 25%) = 1 463 руб.
```

**Senior Go-разработчик, удалёнка, без зарплаты:**
```
1500 × 1.5 (Senior) × 1.6 (дефицит Go) × 1.0 (нет скидки) = 3 600 руб.
```

**Junior QA Manual, Казань, всё заполнено:**
```
1500 × 0.7 (Junior) × 0.8 (избыток кандидатов) × 0.70 (скидка 30%) = 588 руб.
```

## 8. Метрики монетизации

| Метрика | Формула | Цель (6 мес.) |
|---------|---------|---------------|
| **MRR** | Σ всех подписок | 150 000 руб. |
| **ARPU** | MRR / платящих | 6 000 руб. |
| **Конверсия Free→Pro** | Pro / (Free+Pro) | 5–8% |
| **Churn** | Ушедших / всего за мес. | < 5% |
| **LTV** | ARPU / Churn | > 120 000 руб. |
| **Доход бота** | Промо-размещения/мес | 50 000 руб. |
| **CAC** | Расходы на привлечение / новых | < 3 000 руб. |

## 9. Реализация в текущей архитектуре

### 9.1 Новые сущности

```typescript
// Организация (мультитенантность)
interface Organization {
  id: string;
  name: string;
  plan: 'free' | 'pro' | 'business';
  planExpiresAt: Date;
  maxUsers: number;
  maxVacancies: number;
  maxCandidates: number;
  maxMatchingsPerDay: number;
  createdAt: Date;
}

// Квоты пользователя (дневные лимиты)
interface UserQuota {
  id: string;
  organizationId: string;
  date: string; // YYYY-MM-DD
  matchingsUsed: number;
}

// Промо-размещение (бот)
interface BotPromoPlacement {
  id: string;
  vacancyId: string;
  organizationId: string;
  tier: 'promo' | 'premium';
  price: number;           // рассчитанная цена
  gradeMultiplier: number;
  deficitMultiplier: number;
  completenessDiscount: number;
  paidAt: Date;
  expiresAt: Date;
  impressions: number;
  clicks: number;
  applications: number;
}

// Реферальная ссылка
interface Referral {
  id: string;
  referrerId: string;      // кто привёл
  referredOrgId: string;   // кого привёл
  rewardType: 'percentage' | 'free_month';
  rewardAmount: number;
  status: 'pending' | 'paid';
  createdAt: Date;
}

// Разовая покупка
interface Purchase {
  id: string;
  organizationId: string;
  type: 'vacancy_audit' | 'market_report' | 'candidate_export' | 'extra_vacancy' | 'extra_user';
  amount: number;
  createdAt: Date;
}
```
